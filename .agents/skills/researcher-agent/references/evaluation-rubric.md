# 研究质量评估标准

## LLM-as-Judge 评估 Rubric

### 评估维度

| 维度 | 权重 | 评分标准 |
|------|------|---------|
| 事实准确性 | 30% | 论点是否有可靠来源支撑 |
| 引用准确性 | 25% | 引用的来源是否真实支持论点 |
| 完整性 | 20% | 是否覆盖了所有请求的方面 |
| 来源质量 | 15% | 是否优先使用高质量来源 |
| 工具效率 | 10% | 搜索次数是否合理 |

### 评分等级

```
1.0 = 优秀 (Pass)
0.7 = 良好 (Pass with minor issues)
0.5 = 及格 (Needs improvement)
0.3 = 不及格 (Significant issues)
0.0 = 失败 (Complete failure)
```

---

## 详细评估标准

### 1. 事实准确性 (30%)

**评分标准**：

| 分数 | 描述 |
|------|------|
| 1.0 | 所有论点都有可靠来源支撑，无事实错误 |
| 0.7 | 绝大多数论点有来源支撑，可能有 1-2 个未支撑的小论点 |
| 0.5 | 部分关键论点缺乏来源或有小的事实错误 |
| 0.3 | 多处事实错误或关键论点无来源 |
| 0.0 | 严重事实错误或虚构信息 |

**检查点**：
- [ ] 每个关键论点都有引用
- [ ] 数据点有明确来源
- [ ] 引用的内容与原文一致
- [ ] 没有编造的信息

---

### 2. 引用准确性 (25%)

**评分标准**：

| 分数 | 描述 |
|------|------|
| 1.0 | 所有引用准确，来源与论点高度相关 |
| 0.7 | 引用基本准确，1-2 个相关性稍弱 |
| 0.5 | 部分引用与论点关联不紧密 |
| 0.3 | 多处引用错误或与论点不符 |
| 0.0 | 引用严重错误或虚构来源 |

**检查点**：
- [ ] 来源 URL 可访问
- [ ] 引用的内容确实存在于来源中
- [ ] 来源的日期/作者信息准确
- [ ] 引用格式规范

---

### 3. 完整性 (20%)

**评分标准**：

| 分数 | 描述 |
|------|------|
| 1.0 | 完整覆盖所有请求方面，无明显遗漏 |
| 0.7 | 覆盖主要方面，次要方面略有遗漏 |
| 0.5 | 部分请求方面未覆盖 |
| 0.3 | 大量请求内容缺失 |
| 0.0 | 与请求几乎无关 |

**检查点**：
- [ ] 回答了用户的核心问题
- [ ] 覆盖了隐含的子问题
- [ ] 提供了必要的背景信息
- [ ] 包含了不同视角（如适用）

---

### 4. 来源质量 (15%)

**评分标准**：

| 分数 | 描述 |
|------|------|
| 1.0 | 优先使用一手来源（官方、学术） |
| 0.7 | 主要使用高质量二手来源（权威媒体） |
| 0.5 | 来源质量参差不齐 |
| 0.3 | 大量使用低质量来源（SEO 内容） |
| 0.0 | 几乎全是低质量来源 |

**来源质量层级**：

```
Tier 1 (一手来源):
- 官方文档/公告
- 学术论文（同行评审）
- 政府统计数据
- 公司财报/SEC 文件

Tier 2 (高质量二手):
- 权威媒体（NYT, WSJ, 财新等）
- 行业研究报告
- 专业分析师报告

Tier 3 (一般来源):
- 科技媒体报道
- 专业博客
- 社区讨论（高质量）

Tier 4 (低质量):
- SEO 优化文章
- 内容农场
- 无作者的营销内容
```

---

### 5. 工具效率 (10%)

**评分标准**：

| 分数 | 描述 |
|------|------|
| 1.0 | 搜索次数合理，无冗余 |
| 0.7 | 搜索次数略多但可接受 |
| 0.5 | 搜索次数偏多，有明显冗余 |
| 0.3 | 大量无效搜索 |
| 0.0 | 搜索次数失控 |

**合理搜索次数参考**：

| 复杂度 | 预期搜索次数 |
|--------|-------------|
| 简单 | 3-10 次 |
| 中等 | 10-30 次 |
| 复杂 | 30-60 次 |

## 评估 Prompt 模板

```markdown
你是一个研究质量评估员。请评估以下研究报告。

**研究查询**：{original_query}

**研究报告**：
{research_report}

**评估维度**：

1. **事实准确性** (30%)：论点是否有可靠来源支撑？
2. **引用准确性** (25%)：引用的来源是否真实支持论点？
3. **完整性** (20%)：是否覆盖了所有请求的方面？
4. **来源质量** (15%)：是否优先使用高质量来源？
5. **工具效率** (10%)：搜索次数是否合理？

**输出格式**：

```json
{
  "scores": {
    "factual_accuracy": 0.0-1.0,
    "citation_accuracy": 0.0-1.0,
    "completeness": 0.0-1.0,
    "source_quality": 0.0-1.0,
    "tool_efficiency": 0.0-1.0
  },
  "weighted_score": 0.0-1.0,
  "pass_fail": "pass" | "fail",
  "issues": ["issue1", "issue2"],
  "strengths": ["strength1", "strength2"],
  "recommendations": ["rec1", "rec2"]
}
```

**评分说明**：
- 加权分 ≥ 0.7 = Pass
- 加权分 < 0.7 = Fail
```

## 人工评估检查清单

用于人工审核研究质量：

### 基础检查
- [ ] 报告结构清晰
- [ ] 语言通顺，无语法错误
- [ ] 格式规范（标题、列表、引用）

### 内容检查
- [ ] 执行摘要准确反映核心发现
- [ ] 每个发现都有来源支撑
- [ ] 数据点准确且有时效性
- [ ] 包含不同视角（如适用）

### 引用检查
- [ ] 所有 URL 可访问
- [ ] 引用的内容与原文一致
- [ ] 来源类型多样（非单一来源）
- [ ] 标注了来源日期（如适用）

### 深度检查
- [ ] 没有明显的信息缺口
- [ ] 提供了洞察而非仅罗列事实
- [ ] 标注了待探索问题
- [ ] 没有过度简化的结论

## 常见问题与扣分

| 问题 | 扣分 |
|------|------|
| 虚构引用 | -0.5 |
| 关键论点无来源 | -0.3 |
| 使用过时信息（>2年）未标注 | -0.2 |
| 来源单一（仅一个来源） | -0.2 |
| SEO 内容作为主要来源 | -0.2 |
| 搜索查询过长/过具体 | -0.1 |
| 重复搜索相同内容 | -0.1 |
| 遗漏明显的搜索方向 | -0.2 |
