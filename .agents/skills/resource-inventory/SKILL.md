# Resource Inventory Skill

扫描项目，盘点文档，生成带即梦中文提示词的完整资源清单。

---

## 工作流程

按顺序执行以下 6 步。每步完成后再进入下一步。

### Step 1 — 扫描项目

用 Glob 和 Read 工具扫描项目：

1. **扫描目录结构**：用 Glob 检查 `docs/`、`projects/`、根目录的文件列表
2. **读取文档内容**：
   - `docs/` 下所有 `.md` 文件（PRD、IA、视觉方案、架构图等）
   - 根目录下的游戏设计文档、需求文档（通常是大文件，优先读压缩版）
   - `projects/` 下的源代码结构（判断开发阶段用）
3. **检测已有资源**：扫描项目中的图片（`.png/.jpg/.jpeg/.svg/.webp`）、音频（`.mp3/.wav/.ogg`）、字体文件

将扫描结果整理为内部数据，供后续步骤使用。

### Step 2 — 识别项目阶段

根据扫描结果，按以下优先级判断当前阶段：

| 阶段 | 判定条件 | 资源关注重心 |
|---|---|---|
| **概念期** | 仅有设计文档/策划案，`docs/` 为空或仅有早期笔记 | 品牌核心、主视觉、关键角色概念 |
| **规划期** | `docs/` 有 PRD，但无 IA 文档 | 全部美术分类 + UI 框架 |
| **设计期** | `docs/` 有 IA 或视觉方案文档 | 细化到每个具体条目 + 交互元素 |
| **开发期** | `projects/` 下有源代码 | 补充缺失资源 + 动效/特效素材 |
| **测试期** | `tests/e2e/` 有测试用例 | 最终校验 + 收尾替换资源 |

在输出中标注识别到的阶段，并说明判断依据。

### Step 3 — 生成资源清单

从文档内容中提取所有可识别的资源需求，按以下分类组织。每个分类下由粗到细分层：

**1. 品牌资源**
- Logo、标准字体、品牌主色/辅助色、Slogan、宣传封面

**2. 角色美术**
- 主角：立绘、头像、表情差分
- NPC：立绘、头像（按名字逐一列出）
- 其他角色：按文档描述列出

**3. 场景美术**
- 主场景：全景图（复杂场景提供 2-3 个变体）
- 场景细节：近景、特写元素
- 地图/导航图

**4. UI/界面**
- 图标集、按钮、弹窗、HUD 元素、加载画面
- 界面背景图

**5. 道具/物品**
- 收集品、道具、线索物品（逐一列出）

**6. 特效/动效**
- 粒子效果、转场动画、UI 动效参考图

**7. 音频**
- BGM（按场景/情绪分类）
- 音效（按类型分类）

**8. 文案**
- 对话文本、物品描述、教程文案、成就文案

每个条目标注状态：
- `[待创建]` — 项目中没有找到对应文件
- `[已有: 路径]` — 找到了对应文件，列出路径
- `[需更新]` — 文件存在但可能需要更新（文档描述与现有文件不匹配时）

### Step 4 — 美术风格访谈

读取 `references/art-styles-catalog.md` 获取风格选项列表，然后用 AskUserQuestion 向用户询问 3 个问题：

**问题 1 — 整体美术风格**（必问）
提供 4-5 个与项目最相关的风格选项（根据文档内容推测，比如古村题材优先列水墨国风、工笔画等），加上"其他"选项让用户自定义。

**问题 2 — 色调偏好**（必问）
提供选项：暖色明亮、暖色暗沉、冷色明亮、冷色暗沉、中性柔和、高饱和、单色/双色。

**问题 3 — 画面精细度**（必问）
提供选项：极简、中等、精细、超精细。

将用户的选择记录为全局风格参数，用于下一步生成提示词。如果用户选择"其他"，需要追问具体风格描述。

### Step 5 — 生成即梦提示词

为清单中所有**美术类**资源（品牌图、角色、场景、UI、道具、特效）生成中文提示词。

提示词格式：
```
{风格前缀}，{主体描述}，{动作/状态}，{场景/背景}，{色调附加}，{构图}，{精细度附加}
```

**生成规则**：
- 纯中文，适配即梦平台
- 每条 50-120 字
- 从 `references/art-styles-catalog.md` 中获取风格前缀、色调附加、精细度附加
- 同类资源保持风格一致性（相同前缀）
- 复杂场景（如全景、多角色构图）给出 2-3 个变体，标注差异（如"日间/夜间"、"远景/近景"）
- 非美术类资源（音频、文案）不生成提示词，改给风格参考描述

### Step 6 — 输出文件

按 `templates/resource-inventory-template.md` 的格式组装完整清单，写入：
```
docs/{project-name}-resource-inventory-{YYYY-MM-DD}.md
```

- `{project-name}` 从项目目录名或文档标题中提取
- 日期使用当天日期
- 如果已有同日期文件，覆盖更新

写入完成后，在对话中展示摘要：
```
资源清单已生成：docs/xidi-resource-inventory-2026-04-28.md

📊 概览：
  品牌资源：3 条（待创建 3）
  角色美术：12 条（待创建 12）
  场景美术：8 条（待创建 8）
  UI/界面：15 条（待创建 15）
  道具/物品：6 条（待创建 6）
  特效/动效：4 条（待创建 4）
  音频：5 条（待创建 5）
  文案：3 条（待创建 3）
  ──
  合计：56 条 | 已有：0 | 待创建：56
```

## 注意事项

- **大文件处理**：游戏设计文档可能很长。优先读取压缩版或摘要，需要细节时再读取完整版。避免一次性读取超长文档导致上下文溢出
- **增量更新**：如果 `docs/` 下已有资源清单文件，读取它并与新的扫描结果对比，标记新增/已完成/需更新的条目，而非从零重建
- **提示词质量**：提示词需要具体且有画面感。避免「一个角色」这样模糊的描述，应该写「一位身着青衫、手持折扇的年轻书生，面容清秀，眉眼含笑」
- **中文规范**：提示词中的标点使用中文逗号「，」分隔，不使用英文标点
