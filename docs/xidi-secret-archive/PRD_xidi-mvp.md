# 产品需求文档 (PRD) - 西递秘档：明经遗梦 (MVP版)

## 1. 背景与目标 (Background & Goals)
- **业务背景**：本项目为结合西递实景的黑客松项目。旨在通过微信小程序提供轻量级的“游戏化叙事”解谜探索体验，部分替代传统导游，寓教于乐，吸引年轻游客。
- **核心痛点**：传统导游枯燥，而过度重度的游戏化设计又会让人变成“低头族”，影响线下实体观光。
- **业务目标 (MVP阶段)**：
  1. 在极短的黑客松开发周期内，跑通“任务驱动探索 -> LBS触发 -> 极简解谜打卡”的核心体验闭环。
  2. 提供一个视觉上**足够吸引人、风格独特**的地图 UI 界面，展现商业潜力和沉浸感。

## 2. 目标用户 (Target Audience)
- **目标用户**：西递古村落的年轻游客、喜欢新鲜体验的散客以及亲子家庭。
- **核心诉求**：无需复杂的游戏学习成本，希望在游玩中能有一个“神秘向导”引导他们去发现古村落里常人注意不到的细节，获得独特的打卡体验。

## 3. 需求范围 (Scope)

### **In Scope (本次 MVP 必做)**
- **独特的全局沉浸式地图 UI**：弱化传统菜单，以极具设计感的地图作为主视界，留出未来迭代的扩展槽（小百科、任务列表等）。
- **任务驱动型探索机制 (Task-driven)**：系统下发明确的下一步目标（如“前往胡文光牌楼寻找石雕密码”），通过目标带动游客在线下移动，而不是盲目乱逛等待触发。
- **LBS 地理位置触发闭环**：基于 GPS 定位，玩家到达任务指定点附近自动触发剧情和谜题。
- **精简的三大核心关卡闭环**：
  - 序章（线上接取主线任务：寻访明经秘档）
  - 第一幕（胡文光牌楼：寻访与轻量解谜）
  - 终章（观景台/荷花池：集齐印章，揭晓秘档真相）
- **极简解谜组件**：纯前端实现的图片展示、单选题或简单交互。

### **Out of Scope (不涉及/未来做)**
- **长篇大论的小百科系统**：MVP 阶段仅在 UI 上“留出入口位置”，不开发完整的词条库。
- **真实的 AR 增强现实识别**：采用“看实景 + 手机内看图单选”的稳妥方案替代。
- **多分支剧情与完整的8关体验**：黑客松优先验证核心闭环，后续再铺开内容量。
- **复杂的账号、积分、成就、商城系统**。

## 4. 核心功能详述 (Functional Requirements)

### 4.1 沉浸式地图主 UI (Immersive Map UI)
- **模块描述**：小程序首页及唯一常驻主界面。
- **视觉要求**：拒绝死板的标准控件，采用强烈的视觉风格（如国风插画、神秘卷轴风或羊皮纸感）。
- **界面元素**：
  - **当前任务悬浮卡 (Mission Card)**：常驻于页面顶部或底部，清晰指示当前目标（如：“第一幕：前往村口胡文光牌楼”）。
  - **功能扩展位 (Reserved Slots)**：界面边缘优雅地留出【百科】和【背包/印章】的收纳入口，MVP 阶段点击可提示“敬请期待”或仅展示当前收集进度。
  - **地图导航指引**：不提供精准逐级导航，仅标识自身位置和目标建筑的相对方向。

### 4.2 任务派发与 LBS 触发系统
- **触发条件**：
  1. 上一任务完成时，自动派发下一个目标地点的任务卡。
  2. 玩家在现实中移动，手机 GPS 进入目标建筑范围（如 20 米内）。
- **预期行为**：
  - 玩家未到达时，点击任务卡仅展示简短提示语（如“请前往胡文光牌楼下寻找胡老夫子”）。
  - 玩家到达目标坐标，手机震动，主界面自动弹窗进入【剧情/解谜模块】。

### 4.3 剧情与解谜模块 (极简版)
- **触发条件**：LBS 到达目标点后。
- **预期行为**：
  - **剧情对话**：出现虚拟 NPC（如胡老夫子）的立绘和简短对话气泡，交待历史背景。
  - **互动解谜**：通过大图展示局部细节（如牌坊上的动物石雕），让玩家观察线下实物，并在手机上完成单选题或匹配题。
  - **容错机制**：答错两次直接高亮提示正确答案，确保绝不卡关。

### 4.4 收集与结算模块
- **触发条件**：完成某地点的解谜。
- **预期行为**：
  - 播放获得“明经印”（如“仁”字印）的盖章动效。
  - 完成三个关卡后，自动触发终章剧情，展示一段精美的结局文字，并生成一张带有收集印章的成就海报供玩家分享。
