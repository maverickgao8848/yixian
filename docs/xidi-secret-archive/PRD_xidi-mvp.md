# 产品需求文档 (PRD) - 西递秘档：明经遗梦

> **详细设计参考**：完整 NPC 对话、谜题细节、百科全文见 `xidi-secret-archive-dev-version.md`

## 1. 背景与目标 (Background & Goals)
- **业务背景**：本项目为结合西递实景的游戏化叙事解密探索产品。以移动端 Web 应用（后续可转微信小程序）提供"游戏化叙事 + 沉浸式导览 + 轻量收集"体验，部分替代传统导游，寓教于乐，吸引年轻游客。
- **核心痛点**：传统导游枯燥，而过度重度的游戏化设计又会让人变成"低头族"，影响线下实体观光。
- **业务目标**：
  1. 完整交付"普通模式"体验：序章 + 5 幕实地关卡 + 终章，共 7 个节点，主线时长约 1-1.5 小时。
  2. 提供视觉上**足够吸引人、风格独特**的地图 UI 界面，搭配完整的百科、NPC、拍照打卡、道具收集等系统，展现沉浸感与商业潜力。

## 2. 目标用户 (Target Audience)
- **目标用户**：西递古村落的年轻游客、喜欢新鲜体验的散客以及亲子家庭。无需任何专业知识背景。
- **核心诉求**：无需复杂的游戏学习成本，希望在游玩中能有一个"神秘向导"引导他们去发现古村落里常人注意不到的细节，获得独特的打卡体验。

## 3. 需求范围 (Scope)

### **In Scope (本次必做)**
- **沉浸式地图主 UI**：以极具设计感的地图作为主视界，包含探索迷雾、三类地图标记（主线建筑/拍照打卡/特色触发点）、成就系统。
- **7 个主线关卡**：序章（线上）+ 5 幕实地关卡 + 终章，完整剧情对话、谜题逻辑、通关判定。
- **任务驱动型探索机制**：系统下发明确下一步目标，GPS 定位自动触发 + 手动确认双模式。
- **虚拟 NPC 体系**：5 个核心 NPC（胡老夫子、胡文光、胡贯三、胡文照、胡尚熷），立绘 + 对话树 + 音频。
- **极简解谜组件**：四大类型——图像识别、填空作答、点击找茬、机关解谜，每题答错两次自动给提示。
- **百科系统**：10 条百科条目（每条 300-500 字），按通关进度解锁，与谜题联动。
- **拍照打卡系统**：5 处建筑打卡点，地图红色相机图标标注，拍照上传 + 手动确认双模式。
- **道具与收集系统**：3 件初始道具（族谱残页/放大镜/简易罗盘）+ 5 枚明经印 + 2 种特产 + 3 块轶事碎片。
- **2 条支线任务**（绣楼喜事、失落的第六印）+ **2 个隐藏彩蛋**（马头墙的秘密、雨巷丁香）。
- **3 个特色触发点**：2 个特产收集点 + 1 个名人轶事石碑（共 3 块碎片）。
- **分享系统**：通关纪念海报生成 + 社交分享文案。
- **存档系统**：自动保存进度，支持断点续玩。

### **Out of Scope (不涉及/未来做)**
- **真实 AR 增强现实识别**：采用"看实景 + 手机内看图单选"的稳妥方案替代。
- **复杂的账号、积分、商城系统**。
- **多分支剧情系统**：普通模式统一体验，降低理解和开发成本。
- **微信小程序原生开发**：当前阶段使用移动端 Web（React + Vite），后续可能转微信小程序。

## 4. 核心功能详述 (Functional Requirements)

### 4.1 沉浸式地图主 UI (Immersive Map UI)
- **模块描述**：应用首页及唯一常驻主界面。
- **视觉要求**：拒绝死板的标准控件，采用强烈的视觉风格（双生视界：日间空灵卷轴 ⇄ 夜间赛博遗迹）。
- **地图机制**：
  - **探索迷雾**：GPS 定位自动驱散周围迷雾，无需手动操作。
  - **已探索标记**：走过的新巷道自动标记，获得小额积分。
  - **成就系统**：探索 30%/60%/100% 区域分别获得「巷陌行者」「迷宫旅人」「西递通」称号。
  - **离线兼容**：GPS 信号弱时可手动点击「我已到达此处」，但需回答验证性问题。
- **界面元素**：
  - **当前任务悬浮卡 (Mission Card)**：常驻于页面顶部或底部，清晰指示当前目标。
  - **三类地图标记**：①主线建筑（金色图标）②拍照打卡点（红色相机图标）③特色触发点（蓝色星标）。
  - **功能扩展位**：界面边缘留出【百科】和【背包/印章】的收纳入口。
  - **地图导航**：仅标识自身位置和目标建筑相对方向，不提供精准逐级导航，鼓励自主探索。

### 4.2 任务派发与 LBS 触发系统
- **触发条件**：
  1. 上一任务完成时，自动派发下一个目标地点的任务卡。
  2. GPS 进入目标建筑 20 米范围自动触发剧情。
  3. GPS 信号弱时可手动确认到达。
- **预期行为**：
  - 未到达时，点击任务卡仅展示简短提示语。
  - 到达目标坐标，手机震动，主界面自动弹窗进入【剧情/解谜模块】。
- **自由路线**：推荐从村口由南向北逛展，但玩家可自由选择顺序。跳过关卡时 NPC 友善提醒，不强制回头。

### 4.3 剧情与解谜模块
- **四大解谜类型**：

| 解谜类型 | 实现方式 | 应用关卡 |
|---------|---------|---------|
| 图像识别 | 提供高清参考图，玩家选择/点击目标元素 | 瑞玉庭「商」字识别、牌楼石雕观察 |
| 填空作答 | 提供题目，玩家填入答案（支持拼音，同音字判对） | 牌楼形制填空、追慕堂门神填空 |
| 点击找茬 | 高清图内点出错别字位置 | 瑞玉庭错字联 |
| 机关解谜 | 屏幕内虚拟操作（拖拽排序、拼字等） | 孝字拼字、祖先排序、印章排列 |

- **难度控制**：
  - 观察题：可疑区域用微弱闪光提示。
  - 填空题：支持拼音输入，答错两次后给字数提示。
  - 取消数字密码计算、时间限制等高难度要素。
  - 生僻字自动加注拼音和释义。
- **容错机制**：所有谜题答错两次后直接高亮提示正确答案，确保绝不卡关。

### 4.4 百科系统
- **入口位置**：底部导航栏常驻「百科」按钮，任何页面可一键进入。
- **内容规模**：10 条条目，涵盖牌坊建筑（2）、三雕艺术（1）、楹联文化（2）、宗族历史（2）、徽商处世（3）。
- **展示形式**：
  - 已解锁：彩色图标 + 完整文字（300-500 字），支持收藏/分享/朗读。
  - 未解锁：灰色图标 + 「通关第 X 幕后解锁」提示。
- **与谜题联动**：连续两次答题错误时，提示「是否查看相关百科知识？」。

### 4.5 拍照打卡系统
- **5 处打卡点**：每处对应一幕主线建筑（胡文光牌楼、瑞玉庭、敬爱堂、青云轩、履福堂）。
- **地图标注**：红色相机图标，点击弹出简短说明和位置指引。
- **完成状态**：已完成的打卡点图标变绿色，显示「已完成 ✓」。
- **系统自动处理**：拍照后在对联/文字位置叠加说明文字。
- **汇总展示**：「我的相册」页面展示已收集照片，支持生成分享图。

### 4.6 虚拟 NPC 系统
- **5 个核心 NPC**：

| NPC | 身份 | 出现地点 |
|-----|------|---------|
| 胡老夫子 | 胡氏宗族长老，游戏引导者 | 村口牌楼（首次）、敬爱堂、终章观景台 |
| 胡文光 | 明代刺史 | 胡文光牌楼 |
| 胡贯三 | 清代巨商 | 追慕堂 |
| 胡文照 | 清代官员 | 大夫第 |
| 胡尚熷 | 清二品官 | 履福堂 |

- **4 种触发方式**：GPS 围栏触发、手动点击触发、任务进度触发、走近触发（支线）。
- **音频设计**：每人配备 5 句核心语音（总时长 ≤ 2 分钟），古风配乐背景。
- **对话呈现**：立绘 + 音频 + 文字气泡，关键选项影响 NPC 回复但不影响主线走向。

### 4.7 道具与收集系统
- **初始道具**（系统自动发放，对应谜题中自动生效）：
  - 族谱残页：祠堂场景自动高亮关键信息
  - 放大镜：雕刻识别谜题中高亮可疑区域
  - 简易罗盘：方位谜题中显示方向提示
- **明经印 ×5**：通关各关卡获得（仁/义/孝/巧/和），集齐解锁终章。
- **特产道具 ×2**：黄山烧饼 + 黄山毛峰，收集品，集齐触发隐藏成就「徽味行者」。
- **轶事碎片 ×3**：3 个名人轶事石碑触发获得，集齐合成「西递人物志」，解锁称号「明经知音」。
- **隐藏之印**：支线「失落的第六印」中获得，解锁终极称号。

### 4.8 支线与彩蛋
- **支线 1「绣楼喜事」**：大夫第绣楼下 GPS 触发，接住虚拟绣球获得「喜」字印章。不经过不影响主线。
- **支线 2「失落的第六印」**：通关主线后在敬爱堂发现线索，前往水口古桥获得「传承之印」+ 终极称号「明经守护者」。
- **彩蛋 1「马头墙的秘密」**：拍摄 3 张不同形制马头墙上传，获得「墙头行家」徽章。
- **彩蛋 2「雨巷丁香」**：雨天进入应用触发限定 BGM + 「雨巷诗人」徽章。
- **以上所有支线与彩蛋均为可选内容，不参与主线通关。**

### 4.9 收集与结算模块
- **单关结算**：完成解谜后播放「明经印」盖章动效。
- **终章结算**：集齐 5 枚明经印，排列正确顺序后揭晓「明经秘档」真相（胡氏家训五则）。
- **通关奖励**：
  - 「明经有缘人」专属数字徽章
  - 通关纪念海报（可分享）
  - 「我的西递足迹」回顾页面（路线/谜题/道具）
- **额外结算**：根据收集的特产、碎片、支线完成情况，触发额外对话和纪念动画。

## 7. 技术平台说明
- **当前阶段**：移动端 Web 应用，React 18 + TypeScript + Vite + Tailwind + Zustand + Framer Motion。
- **部署方式**：静态托管（Vercel / GitHub Pages），面向移动端浏览器。
- **后续规划**：可能转微信小程序，当前架构保持可迁移性。
